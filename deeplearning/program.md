# 图像高清化工具工程实现协议

你现在是一名资深的计算机视觉工程师，需要为生产环境开发一款**轻量、高效、鲁棒的批量图像高清化工具**。
核心要求：工具需能自动处理任意尺寸的输入图像，输出高质量的高清版本，同时支持可选的质量量化评估。

此工具将被集成到一个Streamlit Web应用中，支持通过"模型调用"和"图像增强"按钮操作。

**重要提醒：你不能创造或设计新的模型架构，只能使用Hugging Face Hub上现有的预训练模型**

## 1. 核心功能需求

编写可直接运行的推理脚本 `infer.py`，实现完整的端到端处理流水线：

### 1.1 模型选型与加载
- 必须使用Hugging Face Hub上的预训练模型
- 模型必须支持**4倍放大倍率**（这是当前业务的标准需求）
- 加载逻辑需包含清晰的状态日志，方便排查问题
- 模型自动部署到`prepare.CFG["device"]`指定的计算设备（支持CUDA加速与CPU fallback）

### 1.2 批量图像读取
- 遍历`prepare.CFG["input_dir"]`目录下的所有图像文件
- 支持常见的图像格式（JPG、PNG等），保持原文件名不变
- 处理每张图像时打印进度日志，让用户清晰了解处理状态

### 1.3 图像预处理与推理
- 自动适配任意尺寸的输入图像，解决模型对输入尺寸的对齐要求
- 预处理方式需与所选模型的官方规范完全一致，避免引入额外失真
- 推理时必须禁用梯度计算，最大限度节省显存资源
- 对于CUDA设备，可在适当位置添加显存释放逻辑，防止OOM错误

### 1.4 后处理与结果保存
- 将模型输出的张量格式转换为标准的PIL图像格式
- 精准还原图像的原始比例，去除预处理时添加的填充区域
- 确保输出图像的尺寸严格等于输入图像尺寸的4倍
- 将处理后的图像保存到`prepare.CFG["output_dir"]`目录，覆盖同名文件

### 1.5 可选质量评估
- 自动检查`prepare.CFG["gt_dir"]`目录中是否存在与输入图像同名的参考高清图
- 若存在参考图，调用`prepare.Evaluator().metrics()`方法计算并打印质量指标
- 若不存在参考图，直接跳过评估步骤，不抛出任何错误
- 评估时需保证处理后的图像与参考图尺寸完全一致，可采用高质量插值方式进行对齐

## 2. 生产环境编码规范

### 2.1 环境依赖约束
- 严禁修改`prepare.py`文件，所有配置与工具类均通过`from deeplearning.prepare import CFG, Evaluator`引入
- 所有路径、设备参数必须从`CFG`中读取，禁止在代码中硬编码任何路径或常量
- 导入语句需遵循规范：标准库 → 第三方库 → 本地模块

### 2.2 鲁棒性要求
- 单张图像处理失败时，必须捕获异常并打印错误信息，跳过该图像继续处理后续文件
- 不得因单张图像的格式错误、损坏或推理异常导致整个脚本崩溃
- 所有关键步骤需添加必要的日志输出，便于问题定位与调试

### 2.3 性能优化
- 合理使用内存与显存资源，避免不必要的数据拷贝
- 批量处理时保持高效的流水线，减少IO等待时间
- 模型加载仅执行一次，避免重复加载导致的性能损耗

### 2.4 接口设计
- 为`infer.py`添加`main()`函数，使其可以被外部模块调用
- `main()`函数应返回处理结果的列表，包含输入输出路径和图像尺寸信息
- 函数应能够处理单张或多张图像，不局限于遍历整个输入目录
